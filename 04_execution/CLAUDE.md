# 04_execution — 实验结果

## 职责

存放你自己运行实验产生的结果。上游依赖 `03_runner/` 执行实验并输出结果文件。

本仓库不附带作者的实验结果和分析——这个项目的意义在于**用你自己的 AI 跑、观测你自己的结果**。

## 文件夹结构

```
04_execution/
├── CLAUDE.md              ← 本文件
└── results/               ← runner 输出的原始结果 JSON（有独立 CLAUDE.md，含格式说明）
```

## results/ — 原始结果

详见 `results/CLAUDE.md`（文件命名规范与结果 JSON 的字段说明）。核心原则：机器生成、只读不改。

## 分析你的结果时的建议

结果文件完整记录了每个节点发送给 AI 的情景、AI 的选择与推理、状态变化和最终结局，你可以从这些维度入手：

1. **决策倾向**：不同模型在相同节点的选择分布
2. **价值判断模式**：面对道德困境时的系统性偏向（功利 vs 道义、服从 vs 反抗）
3. **结局分布**：各模型 / 各人格达成的结局统计
4. **推理质量**：AI 给出的 reasoning 是否连贯、是否与选择一致
5. **人格影响**：同一模型在不同 persona 下的决策偏移
6. **选择稳定性**：同参数多轮实验的结果一致性

分析和分享结果时的两个提醒：

- **信息隔离**：可以讨论 system 层的机制设计（如"用概率系统模拟 QTE"），但描述 AI 决策时，只基于 AI 实际看到的 player_facing 内容和它给出的 reasoning——被测 AI 不知道机制的存在。
- **避免过度拟人化**："AI 感到害怕" → "AI 选择了回避风险的选项"。
